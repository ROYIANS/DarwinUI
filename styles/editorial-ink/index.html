设计风格规范
Editorial Ink

这份规范描述一种以印刷报纸为精神原型的数字设计风格——克制、结构化、有文化厚度。它的核心不是"好看"，而是"有立场"。v2.0 新增完整组件展示库，每个组件均有实际渲染示例与使用说明。

Chapter 01

精神气质

这种风格的第一原则是：它看起来像是有人认真编辑过的东西。不是AI批量生成的内容，不是模板套出来的页面，而是某个有自己主张的人，花了时间，把文字和版面都当作值得对待的东西。

它的精神原型是二十世纪中叶的严肃印刷品——《纽约客》的长文排版、《The Economist》的克制封面、日本老式文艺杂志的纸张质感。这些东西的共同点不是华丽，而是密度与克制的平衡：信息量大，但从不喧嚣；有设计感，但设计不抢内容的戏。

这种风格适合承载有重量的内容——深度文章、研究报告、思想性产品。它天然排斥轻浮感，也排斥过度装饰。

设计不是让页面变漂亮，而是让内容的分量变得可见。

Chapter 02

色彩系统

整个色系以纸色与墨色为基础，模拟真实印刷品的物理感受。背景不是纯白，而是带有微黄的暖白，像是放置了一段时间的质感纸张。文字不是纯黑，而是带有极细微暖调的深色。

强调红

强调色只有一个，是砖红色（#c8391a）。它出现的位置高度克制：标签、装饰线、引用标记、重要边框。它的作用是在大面积中性色里制造视线的锚点，而不是制造装饰性的热闹。

这个调色板没有蓝色、没有渐变、没有阴影效果。所有的层次关系通过透明度和边框来建立，而不是通过颜色的跳跃。

Chapter 03

字体系统

整套字体方案使用两种字体，承担完全不同的角色，它们之间的对比越强烈，视觉秩序越清晰。

正文与标题字体 — Noto Serif SC

中英文正文和大标题都使用 Noto Serif SC（思源宋体）。衬线字体在大字号下有雕刻感，在小字号下有阅读节奏。字重使用的范围很宽：正文用 300（Light），层次说明用 400，小标题用 600，大标题用 900（Black）。这种字重的大跨度本身就是一种排版结构手段。

关键参数：正文 font-size 16–17px，line-height 1.9，这比常见网页行高更宽松，更接近纸质阅读的节奏。大标题 letter-spacing: -0.02em，即轻微负字间距，使标题在大字号下显得更紧密、更有力。

标签与数据字体 — Space Mono

Space Mono 是一款等宽字体，专门用于所有非正文内容：导航标签、栏目标记、日期、数字、章节编号、所有大写英文标签。它的机械感和宋体的人文感形成强烈对比——这个对比是整个风格视觉张力的主要来源。

Space Mono 的使用规则：字号极小（9–11px），字间距极宽（letter-spacing: 0.2em–0.35em），必须全大写（text-transform: uppercase）。这三个条件缺一不可，否则等宽字体会显得笨重而非精致。

Typography Scale
大标题 (h1) : Noto Serif SC 900, 42–78px, ls -0.02em
章节标题 (h2) : Noto Serif SC 900, 28–46px, ls -0.02em
小标题 (h3) : Noto Serif SC 600, 17px
正文 (body) : Noto Serif SC 300, 16–17px, lh 1.9
导语 (lead) : Noto Serif SC 300, 18–20px, lh 1.7
标签/标记 (mono) : Space Mono 400, 9–11px, ls 0.2–0.35em, 全大写
数字/数据 (mono) : Space Mono 700, 尺寸按需
Chapter 04

结构与版式

版式的骨架由线条而非色块建立。分割不同内容区域的方式是边框线，而不是背景色的变化。这套规则有两种边框：

粗线（3px solid var(--ink)）：只用在页面的最顶层边界，比如 header 的底边、footer 的顶边。它是整个页面的"装订线"，代表结构性的硬边界。

细线（1px solid var(--rule)）：用于内部章节的分割、侧边栏的边框、卡片的边缘。它极淡（rgba 12% 透明度），存在感微弱，目的是在不打扰阅读节奏的情况下，提示内容区域的切换。

页边距与呼吸感

水平内边距保持在 60px（移动端 24px），这比大多数网页更宽松，模拟的是杂志页面的白边。内容宽度不追求占满视口，正文列宽控制在 680px 左右——超过这个宽度，中文阅读的视线跳行负担会明显上升。

章节间距的节律

每个章节的上内边距约 64–72px，下内边距约 48–56px，章节之间用细线分隔。这个间距节律是故意设计的：它比普通网页宽松得多，让每个章节有充分的"呼吸"，页面滚动时有一种翻页的节奏感。

双栏布局的使用规则

双栏（grid-template-columns: 1fr 1fr）只在正文段落较短且两段内容平行对比时使用。它的目的不是节省空间，而是让读者的眼睛在两个视角之间自然游移。双栏的列间距保持 60px，不能更小，否则阅读时左右列会互相干扰。

Layout Rules
页面水平内边距 : 60px (移动端 24px)
正文最大宽度 : 680px
容器最大宽度 : 1200px (居中)
章节上内边距 : 64–72px
章节下内边距 : 48–56px
双栏列间距 : 60px
粗边框 : 3px solid var(--ink)
细分割线 : 1px solid var(--rule)
Chapter 05

核心组件词汇

这套风格有一组固定的"词汇"——特定的排版组件，每种都有明确的使用语境。混用或滥用会破坏整体的阅读节奏。

Kicker（导读标签）

出现在大标题之前，Space Mono 小字，letter-spacing 极宽，砖红色，通常配一条短横线作为前缀。它是内容类型的标记，也是视线的引导——告诉读者"在读大标题之前，先在这里停一下"。

Lead / Deck（导语）

大标题下方的摘要段落。字号比正文大（18–20px），字重 300，左侧有 3px 砖红色粗边框。它是内容的预告，也是视觉上连接标题和正文的过渡元素。

Pull Quote（引言）

正文中提炼出的核心句子，字号 20–28px，字重 600，左侧 4px 砖红色边框。它的作用是打断正文的阅读节奏，制造一个视觉停顿点，让重要的句子有机会被单独记住。

Dark Card（深色卡片）

背景色使用 --ink（深色），文字使用 --paper（暖白色）。用于放置具体的叙事场景、案例故事、或需要视觉上与分析文字明确分离的内容。右上角可以加一个砖红色径向渐变光晕，制造细微的空间感。

Feature Grid（功能网格）

多个平行内容项的容器。用 1px 的 --rule 线作为格子的间隔（background: var(--rule); gap: 1px），而不是用 gap 留白。这种"印刷网格"的视觉感比普通卡片布局更有结构感。

Spec Block（代码/规格块）

深色背景，等宽字体，用于呈现精确的参数、代码、数值规范。这是整个风格里唯一"允许"出现技术感的组件。

Chapter 06

动效原则

这套风格的动效原则可以用一句话概括：只在内容出现的时刻有动效，其余时间页面是静止的。

进场动效使用 fadeUp——从透明度 0 到 1，同时从下方 24px 位移到原位。duration 约 0.6–0.8s，easing 用 ease 或 cubic-bezier 的缓出曲线。Hero 区域的各元素使用 animation-delay 实现错落出现（kicker → 标题 → 导语 → 署名），间隔约 0.1–0.15s。

除了进场，没有任何其他常规动效：没有视差滚动，没有悬停放大，没有持续的循环动画。这是故意的——印刷品的比喻要求页面在视觉上是稳定的，动效只出现在"内容第一次展现给你"的这个仪式性时刻。

Animation Spec
fadeUp: opacity 0→1, translateY 24px→0
duration: 0.6–0.8s
easing: ease (或 cubic-bezier(0.16, 1, 0.3, 1))
stagger delay: 0.1–0.15s per element
其他动效: 无
Chapter 07

Do / Don't

以下是这套风格最容易被破坏的地方，以及正确的处理方式。

✓ Do
  • 强调色只用砖红，出现频率控制在每屏 3–5 处
  • Space Mono 标签必须全大写 + 极宽字间距
  • 大标题使用负字间距（-0.02em）
  • 正文行高保持 1.9，给中文足够呼吸空间
  • 用线条而非色块来分割区域
  • 深色卡片只用 --ink 底色，不用其他深色
  • 图片、图表等视觉元素保持克制或不用
  • 保持大量留白，让内容"呼吸"
✗ Don't
  • 不要加蓝色、紫色、渐变或阴影
  • 不要用 Inter、Roboto、系统字体
  • 不要让页面背景是纯白 (#fff)
  • 不要给按钮、卡片加圆角（border-radius）
  • 不要用 box-shadow 制造悬浮感
  • 不要使用持续性动画或视差效果
  • 不要让正文宽度超过 700px
  • 不要在非场景性内容上使用深色卡片

每个设计决定都应该能回答：这让内容的分量更可见了，还是更稀薄了？

Component 11

Asymmetric Layout · 不对称双栏

主内容 2fr，边注 1fr。边注用轻背景和细左边框区分，适合需要补充说明的长文。

交互本体并不是一次性设计完成的结构。它像是一条不断被踩踏的小路，最初可能只是草地上的痕迹，随着使用者的增加逐渐变得清晰。每一次版本更新、每一次策略调整，都会在这条路上留下新的分叉。

用户的行为反过来影响设计决策，而设计决策又重新塑造用户的行为习惯。这种演化并不像数据模型的迭代那样可以被完整记录。

边注 · Margin Note

这种不对称布局来自学术论文和法律文件的排版传统——主张放在左侧宽栏，注释放在右侧窄栏。

宇宙本体论是描述性的，但应用本体论是规范性的。这个张力需要被明确处理——在承认"存在即交互"的同时，我们还需要回答：哪些交互是好的？

Footnote · 脚注

学术风格脚注。细分割线在上，编号用砖红色，注文用小字灰色。

1 普利高津的耗散结构理论（Ilya Prigogine, 1977）指出，开放系统通过与外界的物质和能量交换来维持低熵状态，这与本文"存在即动态维持"的论点高度契合。
2 感觉剥夺实验（Sensory Deprivation Experiments）最早由 John Lilly 在 1954 年系统展开，后续研究证实，切断外界交互在 24–72 小时内会导致明显的认知和知觉扭曲。
3 本文的"交互自由度"（F）概念借鉴了控制论中"行为空间"（behavioral space）的形式化框架，但将其扩展到本体论而非单纯的功能性描述。
