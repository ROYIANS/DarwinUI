System III
01
Chapter 01

精神
气质
SPIRIT

这套风格的精神来自一个信念：版面不是内容的容器，版面本身就是内容。当一个字的字号是另一个字的三十倍，当一行文字横跨整个页面宽度，当"空白"被设计得像一道伤疤——这些排版决策本身在说话，不需要内容来解释。

它的原型是70–90年代的荷兰实验设计——Wim Crouwel 的网格系统、Experimental Jetset 的自我指涉、早期 Emigre 杂志的桌面出版粗粝感。这些设计师相信：美学立场就是政治立场。怎么排版，就是在宣示你对这个世界持有什么态度。

02
Chapter 02

四色
系统
COLOR

这套风格只用四种颜色：暖白、离白、深黑、暴力红。没有灰阶过渡，没有品牌色系，没有渐变。颜色的存在是为了制造冲击，而不是和谐。红色的出现必须像一个打断——不是装饰，是声明。

White #f2f0eb 主背景。不是纯白，微温，避免屏幕刺目。
Off-white #e8e5de 次级背景。用于表头、hover 状态等需要区分的区域。
Ink #0d0d0b 主文字与边框。带极微暖调，比纯黑更有温度。
Red #e3210a 唯一强调色。用于编号、装饰线、关键标签、打断性元素。必须克制。
CSS Variables
--white: #f2f0eb /* 主背景，微暖白 */
--off: #e8e5de /* 次级背景，hover 用 */
--ink: #0d0d0b /* 主文字、边框，暖黑 */
--red: #e3210a /* 唯一强调色，暴力红 */
--rule: rgba(13,13,11,0.12) /* 极淡分割线 */

Chapter 03

三字
体系
TYPE

Bebas Neue：所有大号展示文字的核心。这是一款无衬线全大写字体，线条均匀，没有粗细变化，看起来像是用直尺画出来的——这种机械感是刻意的。字号越大，它的几何纯粹感越强，大到一定程度会开始产生图形性。

IBM Plex Mono：正文与标签。它的等宽特性让页面有一种打字机的节奏感，每个字符占据相同宽度，制造出一种刻意的非人工感——恰好与 Bebas 的展示感形成张力。IBM Plex Sans 作为辅助，用于需要更自然阅读节奏的正文段落。

Chapter Four · 价值论基础 From Description to Norm
04
从描述到规范

宇宙本体论是描述性的，但应用本体论是规范性的。我们还需要回答：哪些交互模式是好的？

Component 17

ANNOTATION · 边注

交互本体并不是一次性设计完成的结构。它像是一条不断被踩踏的小路，最初只是草地上的痕迹，随着使用者的增加逐渐清晰。每一次版本更新都会在这条路上留下新的分叉。

用户的行为反过来影响设计决策，而设计决策又重新塑造用户的习惯。

边注 这种不对称布局来自学术论文的传统——主张在左宽栏，注释在右窄栏。左侧红色边框是唯一的视觉区分手段。

Component 19

DO / DON'T

最容易破坏这套风格的决策清单。

✓ DO
  • 大号 Bebas 数字作为图形元素（描边、反色、跨版心）
  • 字号跨度越大越好——8px 标签和 200px 大字可以同屏
  • 红色只用于打断——编号、装饰线、最强调的标签
  • 边框共享（负边距）让多个元素融为整体网格
  • 空白区域是故意的设计，不是"还没填"
  • Ticker 滚动文字制造低频的背景噪声感
✗ DON'T
  • 不要加圆角——哪怕 2px 也会破坏整体的硬边质感
  • 不要加阴影或渐变——这是另一套语言的语法
  • 不要让红色超过 5% 的页面面积
  • 不要用 Bebas 写中文正文——它只支持拉丁字符
  • 不要让所有字号都"适中"——没有张力就没有这套风格
  • 不要用超过四种颜色——多出来的都是噪音
